00054 /** \page NumericPromotionTraits Numeric and Promotion Traits
00055 
00056     Meta-information about arithmetic types.
00057     
00058     <UL style="list-style-image:url(documents/bullet.gif)">
00059     <LI> \ref NumericTraits
00060          <BR>&nbsp;&nbsp;&nbsp;<em>Unary traits for promotion, conversion, creation of arithmetic objects</em>
00061     <LI> \ref PromoteTraits
00062          <BR>&nbsp;&nbsp;&nbsp;<em>Binary traits for promotion of arithmetic objects</em>
00063     <LI> \ref SquareRootTraits
00064          <BR>&nbsp;&nbsp;&nbsp;<em>Unary traits for the calculation of the square root of arithmetic objects</em>
00065     <LI> \ref NormTraits
00066          <BR>&nbsp;&nbsp;&nbsp;<em>Unary traits for the calculation of the norm and squared norm of arithmetic objects</em>
00067     </UL>
00068     
00069     These traits classes contain information that is used by generic
00070     algorithms and data structures to determine intermediate and result
00071     types of numerical calculations, to convert between different 
00072     representations of arithmetic types, and to create certain important
00073     constants of each type. Thus, algorithms and data structures
00074     operating that need arithmetic operations can be made more
00075     independent from the actual data representation.
00076     
00077     NumericTraits are implemented as template specializations of one
00078     arithmetic type, while PromoteTraits are specialized for a pair of
00079     arithmetic types that shall be combined in one operation.    
00080 */

00082 /** \page NumericTraits template<> struct NumericTraits<ArithmeticType>
00083 
00084     Unary traits for promotion, conversion, creation of arithmetic objects.
00085 
00086     <b>\#include</b> <vigra/numerictraits.hxx>
00087 
00088     This traits class is used derive important properties of
00089     an arithmetic type. Consider the following algorithm:
00090     
00091     \code
00092     // calculate the sum of a sequence of bytes
00093     int sumBytes(unsigned char * begin, unsigned char * end)
00094     {
00095         int result = 0;
00096         for(; begin != end; ++begin)  result += *begin;
00097         return result;
00098     }
00099     \endcode 
00100     
00101     The return type of this function can not be 'unsigned char' because
00102     the summation would very likely overflow. Since we know the source
00103     type, we can easily choose 'int' as an appropriate return type.
00104     Likewise, we would have chosen 'float' if we had to sum a 
00105     sequence of floats. If we want to make this 
00106     algorithm generic, we would like to derive the appropriate return 
00107     type automatically. This can be done with NumericTraits. 
00108     The code would look like this (we use \ref DataAccessors to 
00109     read the data from the sequence):
00110     
00111     \code
00112     // calculate the sum of any sequence
00113     template <class Iterator, class Accessor>
00114     typename vigra::NumericTraits<typename Accessor::value_type>::Promote
00115     sumSequence(Iterator begin, Iterator end, Accessor a)
00116     {
00117         // an abbreviation
00118         typedef vigra::NumericTraits<typename Accessor::value_type>  SrcTraits;
00119         
00120         // find out result type
00121         typedef typename SrcTraits::Promote ResultType;
00122       
00123         // init result to zero
00124         ResultType result = vigra::NumericTraits<ResultType>::zero();
00125     
00126         for(; begin != end; ++begin)
00127         {  
00128             // cast current item to ResultType and add
00129             result += SrcTraits::toPromote(a(begin));
00130         }
00131         
00132         return result;
00133     }
00134     \endcode
00135     
00136     In this example NumericTraits is not only used to deduce the 
00137     ReturnType of the operation, but also to initialize it with the
00138     constant 'zero'. This is necessary since we do not know in general,
00139     which expression must be used to obtain a zero of some arbitrary
00140     type - '<TT>ResultType result = 0;</TT>' would only work if the 
00141     ResultType had an constructor taking an '<TT>int</TT>' argument, and we 
00142     would not even have any guarantee as to what the semantics of this
00143     constructor are. In addition, the traits are used to cast the 
00144     source type into the promote type.
00145     
00146     Similarly, an algorithm that needs multiplication would use the 
00147     return type <TT>RealPromote</TT> and the functions <TT>one()</TT> and
00148     <TT>toRealPromote()</TT>. The following members are defined in

00320 /** \page PromoteTraits template<> struct PromoteTraits<ArithmeticType1, ArithmeticType2>
00321 
00322     Binary traits for promotion of arithmetic objects.
00323     
00324     <b>\#include</b> <vigra/numerictraits.hxx>
00325 
00326     This traits class is used to determine the appropriate result type
00327     of arithmetic expressions which depend of two arguments. Consider
00328     the following function:
00329     
00330     \code
00331     template <class T>
00332     T min(T t1, T t2)
00333     {
00334         return (t1 < t2) ? t1 : t2;
00335     }
00336     \endcode
00337     
00338     This template is only applicable if both arguments have the same
00339     type. However, sometimes we may want to use the function in cases
00340     where the argument types differ. Then we can deduce the appropriate
00341     return type by using <TT>PromoteTraits</TT>:
00342     
00343     \code
00344     template <class T1, class T2>
00345     typename vigra::PromoteTraits<T1, T2>::Promote
00346     min(T1 t1, T2 t2)
00347     {
00348         return (t1 < t2) ? vigra::PromoteTraits<T1, T2>::toPromote(t1) : 
00349                            vigra::PromoteTraits<T1, T2>::toPromote(t2);
00350     }    
00351     \endcode
00352     
00353     In addition, the traits class provide static functions to cast the
00354     arguments to the promote type. For example, if <TT>T1</TT> were <TT>int</TT> and 
00355     <TT>T2</TT> were <TT>float</TT>, the <TT>Promote</TT> type would be <TT>float</TT>. 
00356     The following members are defined in 
00357     <b> <TT>PromoteTraits<ArithmeticType1, ArithmeticType2></TT></b>:
00358     
00359     <table>
00360     <tr>
00361     <td>
00362     <b> <TT>typedef ... Promote;</TT></b>
00363     </td><td>
00364             promote type 
00365     </td></tr>
00366     <tr><td>
00367     <b> <TT>static Promote toPromote(ArithmeticType1 v);</TT></b> 
00368     
00369     <b> <TT>static Promote toPromote(ArithmeticType2 v);</TT></b>
00370     </td><td>
00371         convert to <TT>Promote</TT> type 
00372     </td></tr>
00373     </table>
00374     
00375     PromoteTraits for the built-in types are defined in <b>\#include</b> <vigra/numerictraits.hxx>
